Add three new features to the existing WorkerShield app. Do not rebuild from scratch. Add these features to the existing codebase:

========================================
FEATURE 1 — PDF EXPORT & SHAREABLE TEXT
========================================

After the WORKERSHIELD RESPONSE final card appears, add two buttons below it:

BUTTON 1: "📄 EXPORT PDF"
BUTTON 2: "📋 COPY FULL REPORT"

EXPORT PDF implementation:
- Use the jsPDF library loaded from CDN: https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js
- When clicked, generate a PDF containing:
  * Header: WorkerShield logo text + "CONFIDENTIAL — UNION DOCUMENT"
  * Date and time generated
  * Local/Union and Employer fields (from context)
  * Problem description
  * Each agent output section with agent name as heading
  * Final WorkerShield Response section
  * Footer: "Generated by WorkerShield — For union use only"
- PDF filename: WorkerShield_Report_YYYY-MM-DD.pdf
- Download automatically to device

COPY FULL REPORT implementation:
- Copies entire report as plain text to clipboard
- Format:
  WORKERSHIELD REPORT
  Generated: [date/time]
  Local: [local]
  Employer: [employer]
  
  PROBLEM:
  [problem text]
  
  [Each agent section with heading and content]
  
  WORKERSHIELD FINAL RESPONSE:
  [final response]
  
  ---
  Generated by WorkerShield — Confidential Union Document
- Show "COPIED TO CLIPBOARD" confirmation for 2 seconds

Style both buttons to match existing app design. Place them directly below the final response card.

========================================
FEATURE 2 — INCIDENT LOGGER
========================================

Add a new tab or section to the app called "INCIDENT LOG" accessible from the main navigation.

The Incident Logger must:

1. QUICK CAPTURE FORM (designed for 30-second entry on the shop floor):
   - Date (auto-fills today)
   - Time (auto-fills current time)
   - Location (dropdown: Shipping Dock / Receiving Dock / Pick Line / Cold Storage / Production Floor / Other)
   - Incident Type (dropdown: Near Miss / Injury / Equipment Issue / Safety Violation / Harassment / Discipline / Grievance / Other)
   - People Involved (text field — names optional, can be "anonymous")
   - Brief Description (textarea — 3-4 sentences max)
   - Witnesses (text field — optional)
   - Management Notified? (Yes / No / Not Yet toggle)
   - Report Filed? (Yes / No toggle)
   - SAVE INCIDENT button

2. STORAGE:
   - Save all incidents to localStorage under key "ws_incidents"
   - Each incident gets a unique ID and timestamp
   - Never loses data between sessions

3. INCIDENT LIST VIEW:
   - Show all logged incidents in reverse chronological order
   - Each entry shows: date, type, location, brief description preview
   - Tap to expand full details
   - Color coding: Near Miss = yellow, Injury = red, Equipment Issue = orange, Other = blue

4. EVIDENCE EXPORT:
   - "EXPORT ALL INCIDENTS" button
   - Generates PDF or copies text of all incidents formatted as:
     WORKPLACE INCIDENT LOG
     Unifor Local 1285 — Saputo Dairy Products Canada G.P.
     Generated: [date]
     
     INCIDENT #[ID]
     Date/Time: [date] [time]
     Location: [location]
     Type: [type]
     Description: [description]
     People Involved: [people]
     Witnesses: [witnesses]
     Management Notified: [yes/no]
     Report Filed: [yes/no]
     ---

5. SEND TO WORKERSHIELD button on each incident:
   - Pre-fills the main WorkerShield problem textarea with the incident details
   - Switches to main WorkerShield tab
   - User can then activate agents to analyze the incident

6. DELETE option on each incident with confirmation prompt.

Design the Incident Logger to feel fast and minimal — stewards need to log things quickly before evidence disappears.

FEATURE 3 — PWA INSTALLATION
========================================

Ensure WorkerShield is fully installable as a PWA on Android and iOS.

Add/update these files in the /public directory:

manifest.json:
{
  "name": "WorkerShield",
  "short_name": "WorkerShield",
  "description": "Ontario Union Rep & JHSC Multi-Agent System",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#0a1628",
  "theme_color": "#0a1628",
  "orientation": "portrait",
  "icons": [
    {
      "src": "/icon-192.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "any maskable"
    },
    {
      "src": "/icon-512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any maskable"
    }
  ]
}

Add a service worker (sw.js) in /public with basic caching:
- Cache the app shell (index.html, manifest.json) on install
- Serve from cache when offline with a simple "You are offline — reconnect to use WorkerShield" message
- Update cache on new versions

Register the service worker in index.html:
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js');
}

Generate simple placeholder icons programmatically using canvas — draw a shield shape with "WS" text in gold on navy background — save as icon-192.png and icon-512.png using canvas.toDataURL() or generate them as SVG data URIs embedded in the HTML.

Add a banner in the app that appears ONCE when the user hasn't installed the PWA yet:
- "📱 Add WorkerShield to your home screen for instant access"
- "Add to Home Screen" button that triggers the beforeinstallprompt event
- "Dismiss" button that hides it permanently (localStorage flag)
- Style it as a gold banner at the top of the app

Add these meta tags to index.html head:
<meta name="mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
<meta name="apple-mobile-web-app-title" content="WorkerShield">
<link rel="apple-touch-icon" href="/icon-192.png">
<link rel="manifest" href="/manifest.json">
